# Harness environments resolve Image Assets; Sketches stay pure

Raster-backed **Tone Fields** need an async fetch-and-decode step somewhere, but the Sketch contract and the Shading preparation worker protocol are built on purity: a Sketch is a deterministic function of params, seed, time, and Composition Frame (ADR-0002), and the worker re-derives a tone source from `{sketchId, params, seed, compositionFrame}` alone. We decided asset resolution is a Harness environment concern: the **Image Asset parameter** kind in the Parameter Schema declares which assets a Sketch needs, each environment (Studio main thread, workers, later Node/Remotion) owns async fetch-by-stable-ID plus decode into one core-defined decoded-pixels record, and the tone-source generator receives a synchronous pre-resolved asset lookup. The worker protocol stays identity-only — asset IDs are ordinary param values, so job identity and caching need no raster payload — and `packages/core` stays headless, with raster field factories consuming plain pixel records testable from synthetic arrays.

We rejected an async Sketch contract (each Sketch fetching through an injected loader) because it muddies the pure-function invariant and forces every call site async, and rejected main-thread decode with pixel transfer through the worker protocol because the main thread needs its own decoded copy anyway (Tone reference mode), Node/Remotion still needs a separate loader, and the protocol's identity model would grow a raster payload for no shared benefit.
