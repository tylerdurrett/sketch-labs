# Evidence-integrity correction

The confirmation geometry and hashes were generated by commit `f4aef35`, whose
Worker serialized a constant preparation/solver count after execution. Review
found that this understated only the benchmark work in each production
equivalence proof.

The corrected accounting is:

- measured production: one registered-generator preparation, one solver pass;
- production equivalence: one registered-generator preparation and solver pass,
  followed by one benchmark tuple/target preparation—two preparations and one
  solver pass total;
- injected equivalence: one benchmark preparation and one solver pass.

The harness now increments these counts at the actual production-generate,
benchmark-resolve, and injected-generate call sites. Focused tests assert the
call counts for every execution branch. The two raw records have only their
production-equivalence `preparationCount` corrected from `1` to `2`; no
geometry, identity, target, Scene, diagnostics, timing, capture, or decision
field changed.
